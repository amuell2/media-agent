## Platform Overview

### StreamVerse Platform Components

The StreamVerse platform consists of four core services:

| Service | Purpose | Port |
|---------|---------|------|
| Streaming API | Manages live stream ingestion and delivery | 3002 |
| Broadcast API | Handles broadcast lifecycle and scheduling | 3003 |
| CDN Service | Content delivery and media asset management | 3004 |
| Analytics Service | Real-time and historical analytics | 3005 |

### Key Features

- **Multi-Protocol Ingest**: Support for RTMP, SRT, WebRTC, and RIST
- **Adaptive Bitrate Streaming**: Automatic quality adjustment based on viewer bandwidth
- **Global CDN**: 200+ edge locations for low-latency delivery
- **Real-Time Analytics**: Sub-second metrics for live monitoring
- **Auto-Recording**: Automatic VOD creation from live broadcasts
- **DRM Integration**: Widevine, FairPlay, and PlayReady support
- **API-First Design**: RESTful APIs for complete platform control

## Technical Specifications

### Supported Input Formats

| Protocol | Codecs | Max Resolution | Max Bitrate |
|----------|--------|----------------|-------------|
| RTMP | H.264, AAC | 4K (3840x2160) | 25 Mbps |
| SRT | H.264, H.265, AAC | 4K (3840x2160) | 50 Mbps |
| WebRTC | VP8, VP9, Opus | 1080p | 8 Mbps |
| RIST | H.264, H.265, AAC | 4K (3840x2160) | 50 Mbps |

### Output Formats

| Format | Use Case | Latency |

| LL-HLS | Low-latency playback | 2-5 seconds |
| DASH | DRM-protected content | 10-30 seconds |
